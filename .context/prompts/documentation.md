## Prompt

```
Create/update documentation for:

**Subject**: [What needs documenting]

**Documentation Type**:
- [ ] API reference
- [ ] Architecture overview
- [ ] How-to guide
- [ ] Decision record (ADR)
- [ ] .context/ domain file
- [ ] Code comments
- [ ] README update

**Audience**:
- [ ] New team members
- [ ] External developers
- [ ] AI tools
- [ ] All of the above

**Key Points to Cover**:
1. [Point 1]
2. [Point 2]
3. [...]

Requirements:
- Follow the substrate methodology format
- Include "why" not just "what"
- Add practical examples where helpful
- Consider AI consumption (structured, parseable)
- Link to related documentation

For .context/ files, follow this structure:
1. Overview - What and why
2. Implementation - How it works
3. Decision History - Why this approach
4. Examples - Practical usage
```
